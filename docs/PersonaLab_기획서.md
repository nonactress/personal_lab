## 1. 서비스 개요

**PersonaLab**은 개인 개발자가 실제 사용자 테스트 없이도 타겟 사용자의 UX 반응을 AI로 시뮬레이션하는 자동화 검증 도구다.

소스코드(React, HTML 등)를 업로드하면 한국인 100만 명 페르소나 데이터 기반으로 타겟 사용자가 해당 UI에서 어디서 멈추고, 왜 이탈하는지를 분 단위로 예측한다. 분석 결과는 코드 수정 프롬프트(Fix Prompt)로 자동 생성되어 Cursor, Claude 같은 AI IDE에 바로 붙여넣기 가능하다.

---

## 2. 문제 정의

### 개발자 편향(Developer Bias) 문제

개인 개발자와 소규모 팀은 제품을 만들면서 자신이 곧 타겟 사용자라고 가정하는 경향이 있다. 그러나 개발자는 도메인 지식, 기술 친숙도, 사용 맥락 모두에서 일반 사용자와 다르다.

**결과:**
- 개발자가 "직관적"이라고 판단한 UI에서 실제 사용자는 이탈
- 출시 후 높은 이탈률, 낮은 전환율 → 재작업 비용 발생
- 사용자 테스트는 모집·진행에 최소 수주, 수백만 원 소요 → 개인 개발자에게 비현실적

### 시장 현황

| 구분 | 기존 방식 | PersonaLab |
|------|-----------|------------|
| 사용자 모집 | 필요 (최소 5~10명) | 불필요 |
| 테스트 준비 기간 | 1~3주 | 없음 |
| 비용 | 수십만~수백만 원 | 크레딧 소모 (분당 처리) |
| 결과물 | 인터뷰 녹취, 정성 리포트 | 코드 수정 프롬프트 직접 생성 |
| 대상 규모 | 소수 샘플 | 40개 인구통계 strata × 가중 시뮬레이션 |

## 3. 핵심 가설

| 번호 | 가설 | 검증 방향 |
|------|------|-----------|
| H1 | 개발자는 타겟 사용자 행동을 체계적으로 잘못 예측한다 | `developer_assumption` vs 실제 시뮬레이션 결과 비교 |
| H2 | 행동 파라미터(디지털 리터러시, 빨리빨리 성향, 인내 임계값)가 UX 마찰을 예측한다 | 파라미터 값과 confusion_events 상관 분석 |
| H3 | AI Think-Aloud가 실제 사용자 발화를 근사할 수 있다 | Nemotron 페르소나 기반 think_aloud 생성 품질 검증 |
| H4 | 자동 분석이 실제 UX 마찰 지점과 상관된다 | friction_map과 실제 이탈 지점 일치율 |
| H5 | 개인 개발자는 리소스 부족으로 UX 검증을 체계적으로 건너뛴다 | 타겟 사용자 설문/인터뷰 |

## 4. 솔루션 — AI 파이프라인

### 4단계 자동 분석 흐름

```
[사용자 입력]
소스코드 업로드 + 타겟 사용자 설정 + 태스크 정의
        ↓
[M1 — UI 맵 추출]
코드에서 UI 컴포넌트 자동 파싱
→ 버튼, 입력폼, 네비게이션 요소 + 시각적 위계 + 잠재 이슈
        ↓
[M3 — 행동 시뮬레이션]
Nemotron 페르소나 × 행동 파라미터 적용
→ Think-Aloud 생성 + confusion_events + 이탈 여부
        ↓
[M4 — 마찰 집계 + 수정 프롬프트]
40개 strata 가중 집계 → Friction Map
→ risk_level + top3 이슈 + Fix Prompt 자동 생성
```

### 분석 결과 예시

**Friction Map:** UI 요소별 영향받는 사용자 비율 (0~100%)  
**Risk Level:** critical(출시 위험) / warning(개선 권장) / ok(출시 가능)  
**Fix Prompt:** AI IDE에 바로 붙여넣기 가능한 코드 수정 지시문

## 5. 데이터 기반 — Nemotron-Personas-Korea

**데이터셋:** nvidia/Nemotron-Personas-Korea (HuggingFace)  
**총 페르소나:** 1,000,000명  
**Strata 구조:** 40개 인구통계 조합 (연령 5구간 × 학력 4구간 × 성별 2)

| 인구통계 축 | 구분 |
|------------|------|
| 연령 | 10~20대 / 30대 / 40대 / 50대 / 60대+ |
| 학력 | 고졸이하 / 전문대 / 대졸 / 대학원 |
| 성별 | 남자 / 여자 |

**페르소나 구성 필드:**
- `persona`: 인물 서사 (수백 자 한국어 텍스트)
- `professional_persona`: 직업/일상 패턴
- `hobbies_and_interests`: 취미 및 앱 사용 습관
- `cultural_background`: 문화적 맥락
- `skills_and_expertise`: 기술 역량

**행동 파라미터 (Nemotron → PersonaLab 변환):**
- `digital_literacy`: 학력 + 직종 기반 디지털 친숙도 (0.3~0.85)
- `bballi_bballi`: 연령 기반 빨리빨리 성향 — 젊을수록 높음
- `patience_threshold_sec`: 로딩/지연 허용 임계값(초)
- `trust_disposition`: 서비스 신뢰 성향
- `mental_model_anchors`: 익숙한 앱 기반 UI 기대값 (카카오톡, 유튜브, 넷플릭스 등)

## 6. 타겟 사용자

**주요 타겟:** 1인 개발자 / 2~3인 스타트업  
**상황:** 앱/웹 서비스 출시 전 빠른 UX 검증 필요, 사용자 모집 불가  
**구체 페인포인트:**
- "내가 만든 UI가 실제 40대 주부한테 어떻게 보일지 모름"
- "출시 후 이탈률 높은데 어디서 막히는지 파악 불가"
- "UX 리서치 회사에 맡길 예산 없음"

## 7. 기술 스택

| 레이어 | 기술 |
|--------|------|
| Backend API | FastAPI (Python) |
| LLM | Groq API — llama-3.3-70b-versatile |
| 페르소나 데이터 | nvidia/Nemotron-Personas-Korea (HuggingFace) |
| Frontend | Alpine.js + Tailwind CSS |
| 배포 | Uvicorn (로컬 → 클라우드 이전 가능) |

**API 엔드포인트:**
- `POST /build-cast` — 인구통계 필터로 strata 선택
- `POST /analyze` — 소스코드 분석 + 시뮬레이션 실행
- `GET /health` — 서버 상태 확인

## 8. 차별화 포인트

1. **한국인 특화 데이터** — 빨리빨리 문화, 앱 사용 습관(카카오, 배달, 네이버), 지역별 디지털 격차 반영
2. **코드 직접 입력** — URL 또는 소스파일 업로드로 즉시 분석, 목업 불필요
3. **Fix Prompt 자동 생성** — 분석 결과가 AI IDE 프롬프트로 바로 출력 → 수정까지 자동화
4. **가중 집계** — 40개 인구통계 strata 인구 비례 가중치 적용, 통계적 대표성 확보
