# C1 — 데이터 소스의 다양성

## Think-Aloud란?

사용자가 앱을 사용하면서 머릿속 생각을 소리내어 말하게 하는 UX 연구 방법.

```
실제 예시:
"이 버튼이 뭔지 모르겠는데... 그냥 눌러볼까?"
"아 이게 다음 화면으로 가는 건가? 저장은 됐나?"
"여기서 뒤로 가면 처음부터 다시 해야 하나 무서운데"
```

우리 서비스에서 두 가지 역할:
- **데이터 소스**: 실제 타겟 집단의 Think-Aloud 수집 → 행동 파라미터 보정
- **시뮬레이션 출력**: M3가 페르소나 입장에서 이 내부 독백을 생성하는 게 최종 목표

즉, 실제 사람의 Think-Aloud 데이터가 많을수록 시뮬레이션이 현실에 가까워진다.

---

## 문제: 앱스토어 리뷰의 한계

앱스토어 리뷰는 데이터 채널로서 신뢰도가 낮다:

| 문제 | 이유 |
|------|------|
| 인구통계 불명확 | 누가 썼는지 알 수 없음 |
| 극단값 편향 | 1점/5점만 리뷰 작성, 3점(보통)은 침묵 |
| 비구조화 | "별로예요", "좋아요" 수준의 텍스트 |
| 행동 아님 | 사후 감정이지, 실시간 행동 패턴이 아님 |

→ 앱스토어 리뷰 의존도를 낮추고, **공공 데이터 + 커뮤니티 데이터**로 대체

---

## 데이터 소스 재설계

### 1순위 — 공공 데이터 (신뢰도 최고, 무료)

| 소스                           | 내용                      | 활용 포인트                                       |
| ---------------------------- | ----------------------- | -------------------------------------------- |
| **디지털정보격차 실태조사** (과기정통부, 매년) | 연령/지역/소득별 디지털 역량 수치     | `digital_literacy` 파라미터 직접 근거                |
| **한국미디어패널조사** (KISDI, 매년)    | 기기별 사용 패턴, 연령대별 앱 사용 시간 | `bballi_bballi`, `ui_pattern_familiarity` 근거 |
| **인터넷이용실태조사** (KISA, 매년)     | 인터넷 서비스 이용률 연령별 분류      | 집단별 디지털 경험 수준                                |
| **KOSIS 통계청**                | 인구 구조, 스마트폰 보급률         | 타겟 집단 규모 파악                                  |

→ 이 데이터들은 연도별로 있어서 **C3 시간적 유효성**과도 연결됨

### 2순위 — 커뮤니티 데이터 (실제 불만, 구조화 어려움)

| 소스 | 내용 | 수집 방법 |
|------|------|---------|
| **네이트판 / 보배드림** | 중장년층 실제 앱 사용 경험담 | 키워드 크롤링 |
| **에브리타임** | 20대 대학생 UX 불만 | 크롤링 |
| **유튜브 댓글** | 앱 튜토리얼 영상 댓글 — 실제 막히는 부분 | YouTube Data API |
| **카카오 오픈채팅** | 시니어 스마트폰 교실 등 커뮤니티 | 수동 수집 |

### 3순위 — Think-Aloud 직접 수집 (가장 정확, 비용 높음)

```
1단계 (베타): 베타 테스터 30명
  └─ 실제 Think-Aloud 세션 화면 녹화 + 음성 전사
  └─ 전사 텍스트 → 벡터 DB 저장
  └─ 혼란 발생 지점 태깅 → 행동 파라미터 역추론

2단계 (본선 이후): 누적 데이터로 자동 보정
  └─ 시뮬레이션 결과 vs 실제 테스트 비교
  └─ 차이가 큰 파라미터 자동 업데이트
```

### 4순위 — UX 연구 논문 (검증된 수치, 오래됨)

```
우선 탐색 논문 유형:
  - 노인 디지털 기기 사용 연구 (한국 포함)
  - OCEAN-기술수용 상관 연구
  - 연령별 인지 부하 패턴 연구
  - 한국 사용자 UX 특성 연구
```

## 소스별 신뢰 가중치 (재정의)

```python
SOURCE_WEIGHTS = {
    "think_aloud_direct":       1.0,   # 직접 수집한 실시간 행동
    "public_data_government":   0.9,   # 공공 데이터 (대규모, 검증됨)
    "ux_paper_controlled":      0.85,  # 통제 실험 논문
    "ux_paper_survey":          0.65,  # 설문 기반 논문
    "community_text":           0.45,  # 커뮤니티 (편향 있음)
    "app_store_review":         0.25,  # 극단값 편향, 최후순위
}
```

## 단계별 구축 계획

```
서류 제출 전 (지금)
  → 공공 데이터 다운로드 및 파싱 (디지털정보격차 실태조사 최우선)
  → UX 논문 RAG 구축

예선~본선
  → 커뮤니티 크롤링 추가
  → 베타 테스터 Think-Aloud 수집 시작

본선 이후
  → 시뮬레이션 vs 실제 비교로 파라미터 자동 보정
```
