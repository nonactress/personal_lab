## 문제 정의

### 우리가 직접 겪은 문제

사이드/팀 프로젝트에서 반복되는 상황:
- 테스트해줄 사람이 없음
- 지인은 긍정적 피드백만 줌
- 검증 없이 출시 → 아무도 안 씀

### 시장 데이터

| 지표 | 수치 |
|------|------|
| 국내 스타트업 초기 실패 원인 1위 | 시장 수요 미검증 42% |
| 사용자 검증 없이 출시 경험 (개인 개발자) | 70% 이상 |
| 실제 사용자 섭외 비용 | 평균 3~5일, 10~30만 원 |

---

## 핵심 인사이트: Developer Bias

기존 서비스들의 공통 가정: **"개발자가 자기 타겟을 알고 있다"**

하지만 실제 개발자가 아는 것 vs 모르는 것:

| 아는 것 | 모르는 것 |
|---------|---------|
| 나이, 직업, 성별 | 앱을 어떻게 탐색하는가 |
| 대략적인 라이프스타일 | 어떤 UI 패턴에 익숙한가 |
| | 어느 순간 인내심이 끊기는가 |
| | 처음 보는 화면에서 뭘 먼저 보는가 |

개발자는 인구통계는 알지만 **행동 패턴은 모른다**.
자기가 만든 서비스는 자기가 제일 잘 쓰기 때문에 일반 사용자가 막히는 지점을 못 본다 → **Developer Bias**

## 기존 해결책의 한계

### GPT 직접 활용
- 프롬프트마다 결과 달라 일관성 없음
- 이전 버전과 비교 불가
- 특수 타겟(60대 어르신, 특정 직군)에 부정확

### Uxia (해외 서비스)
- 월 수십만 원 → 개인 개발자 접근 어려움
- 영어권 중심 → 한국어 서비스 검증 한계
- 대형 팀 타겟 → 개인 개발자에게 복잡

## 서비스 개요: PersonaLab

> "개발자가 모르는 타겟 사용자의 행동 패턴을 AI가 대신 분석해서 알려준다"

### 작동 방식

```
1. 개발자 입력
   └─ 서비스 URL + 타겟 설명
      예) "60대 어르신 대상 약 복용 알림 앱"

2. AI 행동 파라미터 생성
   └─ 해당 집단의 실제 행동 패턴 분석
      (기술 이해도, 인내심, 멘탈 모델 등)

3. 페르소나 서비스 탐색
   └─ "왜 이 버튼을 못 찾았는가"
   └─ 내부 독백 + 혼란 지수 실시간 계산

4. 리포트 출력
   └─ 이탈 위험 지점 Top 3
   └─ 개선 액션 아이템
   └─ 이전 버전 대비 변화
```

### 아웃풋 예시

```
📊 시뮬레이션 결과

전체 태스크 완료율 : 34%
평균 혼란 지수     : 67 / 100

🔴 이탈 위험 지점 1위 — 회원가입 화면 (혼란 89)
"이메일 인증이 뭔지는 아는데... 그냥 다음 누르면 안 되나?"

🔴 이탈 위험 지점 2위 — 약 추가 화면 (혼란 76)
"버튼이 너무 작아서 잘 안 눌려"

✅ 개선 액션 아이템
1. 이메일 인증 단계에 한 줄 설명 추가
2. 주요 버튼 크기 48px 이상으로 확대
3. 알림 토글 ON/OFF 텍스트 명시
```

## 기술 구조

```
입력
  └─ URL + 타겟 설명

서비스 분석
  └─ Playwright 스크린샷 + Vision 모델

페르소나 생성
  └─ UX 연구 데이터 RAG + Qwen
  └─ 행동 파라미터 자동 생성

시뮬레이션
  └─ 파라미터 기반 동적 프롬프트
  └─ Chain-of-Thought로 Think-Aloud 생성
  └─ 혼란 지수 계산

출력
  └─ 리포트 대시보드
  └─ 버전별 비교
```

**기술 스택**: Qwen (파인튜닝) + FastAPI + Playwright + Vision 모델 + RAG

## 진화 로드맵

### 1단계: 서류~예선
- UX 연구 논문 + 앱스토어 리뷰로 베이스라인 행동 패턴 구축
- 일반적인 타겟 즉시 분석 가능

### 2단계: 예선~본선
- 베타 테스터 30명 Think-Aloud 수집
- 한국 사용자 특화 패턴 보정
- 특수 타겟 정확도 향상

### 3단계: 본선 이후
- "30대 직장인"만 입력해도 학습된 패턴으로 자동 분석
- 개발자 입력 최소화
- B2B SaaS 확장

## 경쟁사 분석 (Uxia)

### 카피할 베이스라인 기능
1. URL → Playwright 스크린샷 → Vision 모델 분석
2. 인구통계 기반 AI 페르소나 생성
3. Think-Aloud 텍스트 생성
4. 마찰 지점 감지 + 리포트 출력
5. SUS 점수 계산

### 차별화 포인트

| # | 항목 | Uxia | PersonaLab |
|---|------|------|-----------|
| 1 | 행동 파라미터 | 기술 이해도만 | OCEAN 모델 + 인내심/상황 수정자 |
| 2 | 한국 맥락 | 글로벌 | 빨리빨리 문화, 정보밀도, 한국 규제 |
| 3 | 버전 비교 | 없음 | V1→V2→V3 자동 추적 |
| 4 | 가격 | 엔터프라이즈 | 무료 진입 / 건당 과금 |
| 5 | 타겟 | 대형 팀 | 개인 개발자 |

### 포지셔닝
> "한국 사용자의 행동 심리까지 반영한 AI 페르소나로, 개인 개발자도 무료로 UX를 검증하는 서비스"
